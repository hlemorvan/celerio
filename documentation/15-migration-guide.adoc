[[migration-celerio-engine]]
== Migration du moteur Celerio (`/home/herve/celerio/`)

[[jibx-to-jaxb]]
=== JiBX → JAXB (changement majeur)

JiBX 1.2.6 utilisait BCEL pour manipuler le bytecode — incompatible avec le système de modules Java 9+. Le projet JiBX est mort depuis des années.

**Changements effectués** :

1. Suppression des dépendances JiBX (`jibx-run`, `jibx-tools`, `jibx-extras`)
2. Suppression du `maven-jibx-plugin` et du `maven-antrun-plugin` (qui l'invoquait)
3. Suppression de `src/main/config/customization.xml`
4. Ajout de JAXB runtime (`jakarta.xml.bind-api 4.0.2` + `jaxb-runtime 4.0.5`)
5. Annotation de toutes les classes de configuration avec JAXB (`@XmlRootElement`, `@XmlAccessorType`, `@XmlAttribute`, `@XmlElement`, `@XmlElementWrapper`, `@XmlTransient`)
6. Remplacement de `JibxMarshaller` par `Jaxb2Marshaller` dans `applicationContext-celerio.xml`

**Bugs découverts lors du test end-to-end** :

JAXB avec `@XmlAccessorType(FIELD)` affecte directement les champs sans passer par les setters, contrairement à JiBX. Les maps dérivées (annotées `@XmlTransient`) restent vides après désérialisation XML.

[cols="2,1,3", options="header"]
|===
|Fichier
|Bug
|Correction

|`Config.java`
|`ForbiddenClassException` XStream
|Ajout d'une allowlist XStream : `XSTREAM.allowTypesByWildcard(new String[] {"com.jaxio.celerio.**", "java.util.**", "java.lang.**"})`

|`Metadata.java`
|`tablesByName` vide après JAXB
|Ajout `afterUnmarshal()` qui appelle `setTables(tables)` et reconstruit les maps de chaque table

|`Table.java`
|`columnsByName` vide après JAXB
|Ajout `afterUnmarshal()` → `setColumns()`, `setIndexes()`, `setImportedKeys()`

|`EntityConfig.java`
|`columnConfigByColumnName` vide après JAXB
|Ajout `afterUnmarshal()` → `setColumnConfigs(columnConfigs)`

|`Celerio.java`
|`sharedEnumConfigs` non chargés
|Correction `@XmlElement(name = "sharedEnumConfig")` (était `"enumConfig"`)
|===

[[spring4-to-spring6]]
=== Spring 4.x → Spring 6.x

Spring 4.x ne supporte pas Java 17+. Spring 6.x exige Java 17 minimum.

**Changements** :
* Version Spring : `4.2.2.RELEASE` → `6.2.7`
* Schémas XML Spring : `spring-beans-4.2.xsd` → sans version (`spring-beans.xsd`)
* Suppression de `spring-context-support` + `aspectjweaver` (LTW obsolète)
* Suppression de `VelocityEngineFactoryBean` (supprimée dans Spring 5) → configuration directe via `@Bean` dans `CelerioConfig.java`
* Remplacement `JibxMarshaller` → `Jaxb2Marshaller`

[[velocity-migration]]
=== Velocity 1.6.2 → velocity-engine-core 2.4.1

* Mise à jour des clés de configuration Velocity 2.x (noms de propriétés renommés)
* `StringUtils.normalizePath` → `StringUtils.normalize`
* Suppression de `Log4JLogChute` → logger SLF4J natif
* log4j 1.x supprimé → logback-classic 1.5.18
* SLF4J 1.5.10 → 2.0.16

**Important pour les templates** : Les variables `$velocityCount` et `$velocityHasNext` ont été supprimées en Velocity 2.x. Voir <<velocity2-changes>> pour les remplacements.

[[spring-security6]]
=== Spring Security 6+ : `{noop}` prefix et `WebSecurityCustomizer`

**Template `UserDetailsServiceImpl.java.p.vm`** : ajout du préfixe `{noop}` pour les mots de passe hardcodés (fallback sans entité Account) :

[source,velocity]
----
if ("user".equals(username)) {
    String password = "{noop}user";
    // ...
}
if ("admin".equals(username)) {
    String password = "{noop}admin";
    // ...
}
----

**Template `SecurityConfiguration.java.p.vm`** : migration complète de l'API Spring Security :
* Suppression de l'héritage `WebSecurityConfigurerAdapter`
* `@EnableGlobalMethodSecurity` → `@EnableMethodSecurity`
* `configure(WebSecurity)` → bean `WebSecurityCustomizer`
* `configure(HttpSecurity)` → bean `SecurityFilterChain`
* `antMatchers` → `requestMatchers`

=== springfox 2.x → springdoc-openapi 3.0.0

[cols="2,2", options="header"]
|===
|Avant (springfox)
|Après (springdoc)

|`springfox-swagger2` + `springfox-swagger-ui`
|`springdoc-openapi-starter-webmvc-ui`

|`@ApiOperation`, `@ApiParam`
|Annotations standard OpenAPI 3 ou autodétection

|URL Swagger UI : `/swagger-ui.html`
|URL Swagger UI : `/swagger-ui/index.html`

|URL API docs : `/v2/api-docs`
|URL API docs : `/v3/api-docs`
|===

Endpoints à ignorer dans `WebSecurityCustomizer` :
[source,java]
----
.requestMatchers("/v3/api-docs/**", "/swagger-ui/**", "/swagger-ui.html")
----
