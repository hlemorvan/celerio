[[bootstrap-real]]
= 4. Démarrer un projet réel avec Celerio

== Créer un nouveau projet (sans base de données existante)

*Prérequis :*

* link:https://adoptium.net/[JDK 25+]
* link:https://maven.apache.org/download.cgi[Maven 3.9+]

Si vous voulez créer un nouveau projet avec Celerio sans base de données existante, suivez simplement le wizard comme expliqué au chapitre <<bootstrap-sample>>. Quand on vous demande de sélectionner un schéma de base exemple, sélectionnez le plus simple (nommé `empty`).

[source,bash]
----
export JAVA_HOME=/home/herve/jdk-25.0.2+10
mvn com.jaxio.celerio:bootstrap-maven-plugin:bootstrap
----

Pour ajouter de nouvelles entités, éditez simplement le fichier SQL exemple généré dans le dossier `quickstart-conf/` et relancez :

[source,bash]
----
export JAVA_HOME=/home/herve/jdk-25.0.2+10
mvn -Pdb,metadata,gen generate-sources
----

== Créer un nouveau projet (depuis une base de données existante)

WARNING: **N'UTILISEZ PAS UNE BASE DE PRODUCTION.** Vous pourriez perdre des données.

=== Suivre le wizard

Suivez le wizard comme expliqué au chapitre <<bootstrap-sample>>. Quand on vous demande de sélectionner un schéma de base exemple, sélectionnez le plus simple (nommé `empty`).

[source,bash]
----
mvn com.jaxio.celerio:bootstrap-maven-plugin:bootstrap
----

Une fois terminé, **ne lancez aucune commande** mis à part `cd <your_app>`.

=== Supprimer le fichier SQL exemple

Pour éviter toute surprise, **supprimez ou renommez** le fichier SQL exemple généré dans `quickstart-conf/`. Nous ne voulons pas l'exécuter par erreur puisque vous allez utiliser votre propre base de données.

=== Éditer pom.xml

Ouvrez le fichier `pom.xml` généré et renseignez toutes les informations pour accéder à votre base de données.

[IMPORTANT]
====
Pour H2 v2 (utilisé depuis celerio-angular-quickstart 2026), l'URL JDBC **ne doit pas** contenir `MVCC=TRUE` ni `FILE_LOCK=NO`. Ces options ont été supprimées dans H2 2.x.
====

Exemple pour H2 v2 :

[source,xml]
----
<jdbc.groupId>com.h2database</jdbc.groupId>
<jdbc.artifactId>h2</jdbc.artifactId>
<jdbc.version>2.3.232</jdbc.version>

<jdbc.driver>org.h2.Driver</jdbc.driver>
<jdbc.url>jdbc:h2:${project.build.directory}/db/mydbname</jdbc.url>
<jdbc.user>root</jdbc.user>
<jdbc.password>manager</jdbc.password>
<jdbc.catalog></jdbc.catalog>
<hibernate.dialect>org.hibernate.dialect.H2Dialect</hibernate.dialect>
<sql-maven-plugin.delimiter>;</sql-maven-plugin.delimiter>
----

Exemple pour MySQL 8+ :

[source,xml]
----
<jdbc.groupId>com.mysql</jdbc.groupId>
<jdbc.artifactId>mysql-connector-j</jdbc.artifactId>
<jdbc.version>9.2.0</jdbc.version>

<jdbc.driver>com.mysql.cj.jdbc.Driver</jdbc.driver>
<jdbc.url>jdbc:mysql://127.0.0.1:3306/angulardb</jdbc.url>
<jdbc.user>root</jdbc.user>
<jdbc.password>manager</jdbc.password>
<jdbc.catalog></jdbc.catalog>
<hibernate.dialect>org.hibernate.dialect.MySQLDialect</hibernate.dialect>
<sql-maven-plugin.delimiter>;</sql-maven-plugin.delimiter>
----

NOTE: Le groupId de MySQL a changé : `mysql:mysql-connector-java` → `com.mysql:mysql-connector-j` depuis la version 8.0.31.

*Conseils :*

* Vous devrez peut-être link:miscellaneous.html[installer manuellement un driver JDBC] non disponible sur Maven Central.
* Vous devrez peut-être accorder des permissions « Read metadata » à votre utilisateur de base de données.

=== Inverser votre base de données

Lancez l'extraction du schéma jusqu'à ce que ça fonctionne :

[source,bash]
----
export JAVA_HOME=/home/herve/jdk-25.0.2+10
mvn -Pmetadata generate-sources
----

En cas de succès, vous devriez obtenir :

[source]
----
[INFO] Write metadata.xml file to src/main/config/celerio-maven-plugin/metadata.xml
[INFO] BUILD SUCCESS
----

Pour satisfaire votre curiosité, vous pouvez ouvrir le fichier `metadata.xml` généré... mais **ne le modifiez pas** — il est utilisé par Celerio pour générer le code source.

=== Générer le code source

Une fois que l'extraction fonctionne, générez le code source :

[source,bash]
----
export JAVA_HOME=/home/herve/jdk-25.0.2+10
mvn -Pgen generate-sources
----

=== Pour aller plus loin...

Ce n'est qu'un début. Vous pouvez personnaliser la génération de 3 façons :

* en modifiant la <<configuration-file,configuration>>
* en suivant les <<conventions,conventions>>
* en créant ou modifiant les <<writing-templates,templates de génération de code>>

== Points d'attention H2 v2

[IMPORTANT]
====
H2 2.x a introduit des changements incompatibles par rapport à H2 1.x :

* `USER` est un mot réservé — le mettre entre guillemets doubles : `"USER"`
* Syntaxe d'identité : `INT NOT NULL IDENTITY` → `INT NOT NULL GENERATED BY DEFAULT AS IDENTITY`
* Supprimer `MVCC=TRUE;FILE_LOCK=NO` de l'URL JDBC
* La vue `INFORMATION_SCHEMA.COLUMNS` a changé de structure (colonne `TYPE_NAME` disponible différemment)
====
